    原理


    deployer 原理是拉取式的自动部署工具

    1. 主要是本地环境通过ssk-key向目标主机发送远程命令
    2. 目标主要接收到远程命令之后通过ssh-key/password向git 代码托管主机拉取代码

    安装deployer

    Liunx 及 MacOS 使用以下命令安装

    curl -LO https://deployer.org/deployer.phar   # 下载phar包
mv deployer.phar /usr/local/bin/dep           # 移动到当前用户的环境变量
chmod +x /usr/local/bin/dep                   # 赋予执行权限

    windows可以使用composer 安装，当然项目里面想要有说法提示，也可以使用composer方式安装

    composer require deployer/deployer         # 使用composer安装

    配置ssh-key

    配置ssh-key之前先来了解一下以下两个概念，以下两个主机之间的通信均使用ssh-key进行安全通信

    • 目标主机: 需要部署代码的目标机群
    • 代码托管主机: 项目代码所在的外网主机
    • 本地环境: 用户的本机环境
    1. 本地环境 <==> 代码托管主机 及 目标主机 <==> 代码托管主机 的ssh-key配置，可参考管网,或者百度 gitlab配置ssh-key
    2. 本地环境 <==> 目标主机配置免密码登陆，原理是把本地环境的公锁加入到目标信任列表当中，可使用以下命令直接添加
        su {local_user} && ssh-copy-id -i ~/.ssh/id_rsa.pub remote_user@remote_server
    #这里需要把关键的变更替换为实际变更 {local_user}=本地用户，  remote_user=目标主机的用户 ,remote_server = 目标主机的ip

    自动部署脚本的编写

    • 新建deploy.php 文件，或者 deploy init 按提示输入相应的关键参数奕可

    <?php
namespace Deployer;
//加载yii2高级版的配置
require 'recipe/yii2-app-advanced.php';
// 项目名称
set('application', 'mini program');
// 代码托管仓库的地址
set('repository', 'git@gitlab.fffff.com:fffff/wechat.wismall.com.git');
// [Optional] Allocate tty for git clone. Default value is false.
set('git_tty', true);
//保留的历史版本
set('keep_releases', 5);
// 版本之间的共享文件及文件夹,一般为数据库配置文件或者本地环境变量
add('shared_files', [
    'api/config/main-local.php',
    'api/config/params-local.php',
]);
add('shared_dirs', [
    'api/runtime',
    'api/web/assets',
    'api/web/upload',
    'backend/web/upload',
    'frontend/web/upload',
]);
//指定 web server 可写的目录
add('writable_dirs', [
    'api/runtime',
    'api/web/assets',
    'api/web/upload',
    'backend/web/upload',
    'frontend/web/upload',
]);
// 目标主机(测试环境)
host('ip')
    ->user('www') //使用的远程登陆的用户
    ->stage('test') // 在test生效
    ->set('deploy_path', '/home/wwwroot/auto-deploy/wechat.wismall.com');

    ->user('www')
    ->stage('dev')

// 目标主机(线上生产环境)

    ->stage('product')
